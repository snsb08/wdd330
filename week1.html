Section 3: Remembering User's Info

Communication between the user and the app or website needs to be mutual to be true communication. We use Input and output for that.
Variables are like label boxes in a kitchen. To be efficient, things need to be in the right container with the right name.

Master Mobile UX Videos:

Do Mobile first! (when the project is not specifically going to be used for desktop). A responsive website does not mean that you made the mobile first. Mobile first give better experience across many devices. Start small and scale up, this addresses constraints of mobile size/ bandwidth.

Reachability matters. Make people's hands able to reach what they need most easily by respecting the areas of the phone that are reachable to the fingers holding a device. The bottom of the page is easier to reach.

Speed matters. People want websites that are easy to use, fast, effective. A web developer's computer could be fast but most peoples' are probably slower. Remove unused code and minimize CSS and JS files. Use image optimization.

Be empathetic to user's needs when they use your website. Not everyone is technological savvy so make things easy and simple to understand. If people like your app, they will be more loyal to it instead of looking for another one.

Make forms as simple as possible. If a lot of info is needed, do pagination on the form. You could use Float labels! Credit card information can be done in one single line. Make phone numbers clickable to call instead of text only.

Pay attention on the way people respond to technology. Do not use pop-ups/pop-overs (true, I skip websites like that myself). If adds are needed, make them as unobtrusive as possible. I like the add underneath the content peeking through a small window. How did they do that? I want to learn that.
