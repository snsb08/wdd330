Typeof are special operators for finding out the type of a value:

typeof 'hello'
<< 'string'

typeof 10
<< 'number'

typeof true
<< 'boolean'

typeof { ninja: 'turtle' }
<< 'object'

typeof [ 1, 2, 3 ]
<< 'object' (All non-primitive data types return a type of 'object'.)

To see the value of a variable, simply enter it in the console and press enter.
Multiple variables can be assigned at the same time if they are separated by commas:

let x = 3, y = 4, z = 5;

Scope: where the variable is visible

  1. Global Scope: variable declared outside of a block which is accessible everywhere in the program. Needs to be kept at minimum specially in long codes or when working on a team. (global do not use "let" or "const")
  2. Local Scope: variable declared inside of a block using "let" or "const" which is not accessible outside of that block.

Non-primitive values: they are assigned by reference, so any changes will affect all references to that object.

const c = { value: 1 };
let d = c; // c.value = 1, d.value = 1
d.value = 2; // c.value = 2, d.value = 2

In the example above, the change to the value property of d also results in the value property of c changing as well. This is because the variables c and d are both referencing the same object, so any changes to one of them will also affect the other.

Backlashes are used to escape special characters in strings:

  • Single quote marks \'
  • Double quote marks \"
  • End of line \n
  • Carriage return \r
  • Tab \t
  • Backslash also needs another backslash \\

If you want to know which character is at a certain position, you can use the charAt() method.

  • "" // double quoted empty string literal
  • '' // single quoted empty string literal
  • `` // empty template literal
  • 0
  • -0 // considered different to 0 by JavaScript!
  • NaN

Try to avoid Type coercion.

Number.isNaN() is used to check if a value is NaN.

Always use hard equality (===) when testing if two values are equal instead of using the soft equality (==).
Always use hard inequality (!==) for a more reliable result when checking that two values are not equal.

Chapter 3 Arrays, Logic, and Loops

Arrays = []
When removing a value from an array using "delete", the space that value occupied is still there as "undefined". The array still has the same number of elements. If you want to remove it completely use the splice() method with a length of 1 and without specifying any values to add.

Splice() can insert or remove values from an array permanently.
ArrayName.splice(index#, lengthRemoved, 'value');
Slice() creates a subarray to display just a slice of the original array.
Sort() is done alphabetically even the numbers are sorted by digit rather than numerically: [10, 5, 9].

Questions: what are the differences between array and sets? Are sets what we use in JSON?

To see the values of an array without duplicates, convert the array to a set.
Sets don't show duplicates of items or even characters.

Maps are similar to 'dictionaries' in other programming languages. Maps can use any data type as a key.

if (condition) {
// code to run if condition is true
} else {
// code to run if condition is false
}

Ternary Operators: shorthand of if/else statement. Just be careful using it as it may make the code harder to read.
condition ? (//code to run if condition is true) : (//code to run if condition is false)

Example:

const n = 5;
n%2 === 0 ? console.log('n is an even number') : console.log('n is an odd number');
<< 'n is an odd number'

Shorter using template string:

console.log(`n is a ${(n%2 === 0)? 'even' : 'odd'} number`);

Switch:

if (number === 4) {
console.log('You rolled a four');
} else if (number === 5) {
console.log('You rolled a five');
} else if(number === 6){
console.log('You rolled a six');
} else {
console.log('You rolled a number less than four');
}

Using Switch:
switch (number) {
case 4:
console.log('You rolled a four');
break;
case 5:
console.log('You rolled a five');
break;
case 6:
console.log('You rolled a six');
break;
default:
console.log('You rolled a number less than four');
break;
}

Always finish a case block with a break.
Default keyword= code if none of the other cases are true.

Chapter 4 Functions

Functions are like mini programs.

Be consistent on your style of writing functions declarations or expressions.
Remember DRY = Don't Repeat Yourself
Always put default parameters after all the other parameters.

Arrow Functions: Are always anonymous, so if you want to refer to them, you must assign them to a variable. They work better for short, anonymous functions.

Function Hoisting: Function declarations are automatically hoisted so they can be used before being defined. Function expressions can only be used after being defined not before.
