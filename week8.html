Chapter 8 Transforms and Transitions

Transform is used to translate, rotate, scale and/or skew any element on the page. We can manipulate an element's appearance using transform functions.

Translation functions allow you to move elements left, right, up, or down. A translated element can only be moved relative to its current position.
Translate (x,y) function moves an element x from the left, and y from the top.
Scale (x,y) function scales an element by the defined factors horizontally then vertically.

Rotate() function rotates an element around the point of origin by a specified angle value. By default, the point of origin is the element's center.
Angles are declared in degrees:
-positive degrees move clockwise
-negative degrees move counterclockwise

Skew(x,y) function specifies a skew along the x and y.

The transition-duration property set how long the transition will take to go from the default to the transitioned state.

The transition-timing-function lets you control the pace of the transition in more detail.
The most common timing functions include: ease, linear, ease-in, ease-out, or ease-in-out.
Try using slow motion to test the animation to really see what is happening before speeding it up to the right timing.

The transition-delay property is used to introduce a delay before the transition begins.

The properties of a transition can be written all in one line divided by space. In the shorthand, if a delay is included, the duration must precede the delay. You can also use the shorthand to specify multiple transitions at different durations, delays and timing functions by separating then with a comma.

Animations

Animations in CSS are controlled by keyframes which are snapshots that define a starting or end point of any smooth transition.
CSS animations are great for subtle, simpler or smaller animations as opposed to JavaScript Animations.

Animation Properties

  • Animation-name attaches an animation to an element
  • Animation-durationdefines the length of time of the animation (in seconds or milliseconds)
  • Animation-timing-function determines how the animation will progress over its duration. (ease, linear, ease-in, ease-out, ease-in-out, cubic-bezier, step-start, step-endâ€¦)
  • Animation-iteration-countdefines how many times the animation will play through or infinite for endless repetition. The default is 1.
  • Animation-directioncan be set to reverse to make the animation go backwards, or to alternate to make it go from forwards to backwards.
  • Animation-delayset a delay to the start of the animation in milliseconds or seconds
  • Animation-fill-modedefines what happens before the first animation iteration begins and after the last animation iteration concludes.
  • Animation-play-statedefines whether the animation is running or paused.

Chapter 12 Canvas, SVG, and Drag and Drop

Canvas: can be used to draw any shapes and patterns using JavaScript.
First, you add a canvas element to the page and add an id name.
By adding a border to the canvas, we can see exactly where it is and the size.

We can use strokeStyle or fillStyle to color the canvas or a shape. First define the stroke color, gradient or pattern, then draw the shape needed.

fillRect and strokeRect methods draw a rectangle

The top-left corner of the canvas is where we have the coordinates (0,0).

CanvasPattern method can make a shape using an image as background.

CanvasGradient method make a gradient effect on the background using either createLinearGradient() or createRadialGradient().
AddColorStop() is used to add color to the gradient, it is called as many times as different colors needed.
Paths create a blueprint for lines, arcs and shapes, but they need stroke color to be seen.

SVG = scalable Vector Graphics is an XML file format.
XML = eXtensible Markup Language. A system meant to annotate text.

Canvas vs SVG:
Canvas allows for pixel manipulation. Canvas operates as immediate mode which means that after you finish drawing a shape, you can't modify it, you would have to draw a new shape. Canvas doesn't have access via the DOM, which makes it faster. Canvas is a better choice for game design with lots of animations.

SVG is retained mode which preserves the image in the XML document, so it can be accessed and changed later. But SVG images can't manipulate pixels like Canvas.

Drag and Drop API allows specific elements to be draggable.

  1. Set the draggable attribute on any HTML elements
  2. add event listener for the dragstart event on the element
  3. Add an event listener for the dragober and drop events on any elements that you want to have accept dropped items

DataTransfer lets us define:

  1. The type of data we're saving of the draggable element
  2. the value of the data itself

Dragover fires when you drag an item over an element.
Drop fires when you drop an item on it.
